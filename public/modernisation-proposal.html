Key gains from the rebuild

Security

Prisma ORM eliminates SQL injection. NextAuth handles sessions securely. TypeScript catches bugs at compile time.

Mobile-ready

Tailwind + responsive layouts replace fixed-width HTML tables. Dispatchers and drivers can use tablets and phones.

Performance

Next.js Server Components stream data. API routes replace page reloads. Partial updates without full-page refresh.

Maintainability

One codebase in Git replaces 30+ file-name-versioned backups. Components are reusable and independently testable.

API-first

Clean REST API means a driver mobile app or customer-facing booking portal can be added without rebuilding the backend.

Data preserved

Existing MySQL database is kept and migrated. All historical records, customers, and companies carry over.

What the codebase revealed:

The app is a PHP/MySQL dispatching system for an accessible transportation company (GTA Accessible) with 9 core modules — the largest being a 76KB monolithic dispatch.php that mixes SQL, HTML, business logic and jQuery all in one file. There are 30+ time-stamped backup copies of files (dispatch_20180224.php, cars_181018.php etc.) sitting in the production folder, which is a significant security and maintainability risk. All database queries use raw string interpolation — no prepared statements — which creates SQL injection exposure across the entire app.

The proposed stack:

  • Next.js 14 (App Router) for both the frontend and API layer
  • TypeScript throughout for type safety and tooling
  • MySQL preserved — the existing data migrates over, Prisma ORM sits on top of it
  • NextAuth.js for session management and role-based access (admin / dispatcher / driver)
  • Tailwind CSS + shadcn/ui for a responsive, mobile-friendly UI
  • React-PDF or Puppeteer to replicate the existing PDF report generation

The migration is structured in 4 phases spanning roughly 8-10 weeks: foundation + auth → dispatch core + customer/vehicle management → reports/PDFs → data migration and cutover.

The key advantage is that the existing MySQL database is kept intact — all historical records, customers, dispatch history and companies carry over. We're replacing the application layer, not the data layer.
